NEW FILES CREATED:

1. build.sh (Linux/Mac)
   - Build compiler with options
   - Supports --debug, --release, --clean
   - Auto-checks for Rust installation

   - Proper Windows error handling

3. run.sh (Linux/Mac)
   - Run compiler with any command
   - Auto-builds if not built
   - Supports --debug flag
   - Passes all arguments to compiler

4. run.bat (Windows)
   - Windows version of run script
   - Same features as run.sh
   - Proper argument handling

5. run-web.sh (Linux/Mac)
   - Start web dashboard
   - Shows endpoint information
   - Auto-builds if needed
   - Professional startup banner

6. run-web.bat (Windows)
   - Windows version of web dashboard runner
   - Same features as run-web.sh
   - Proper Windows console handling

USAGE EXAMPLES:

  Windows:   build.bat --clean --release

                        TESTING PROCEDURES
================================================================================

TO TEST WEB DASHBOARD:

1. Build the project:
   cargo build --release

2. Start web dashboard:
   cargo run --release --bin web-dashboard
   OR: ./run-web.sh (Linux/Mac) or run-web.bat (Windows)

3. Open browser to: http://127.0.0.1:8080
   - Should see professional HTML dashboard
   - Should display Banking Enterprise Edition v2.0
   - Should show all banking features
   - Should list all API endpoints

4. Test API endpoints:
   curl http://127.0.0.1:8080/version
   curl http://127.0.0.1:8080/status
   curl http://127.0.0.1:8080/features
   curl -H "Authorization: Bearer SECRET_TOKEN" http://127.0.0.1:8080/report

5. Verify JSON responses:
   - All endpoints should return valid JSON
   - Status should show "healthy"
   - Version should show 2.0
   - Features should list all banking features
   - Report should return compilation data

TO TEST BUILD SCRIPTS:

1. Test build script:
   Linux/Mac: ./build.sh
   Windows:   build.bat

2. Verify build output:
   - Should compile successfully
   - Should create target/release/corn-compiler
   - Should show next steps

3. Test with options:
   ./build.sh --debug
   ./build.sh --clean --release
   ./build.sh --help

TO TEST RUN SCRIPTS:

1. Test run script:
   Linux/Mac: ./run.sh version
   Windows:   run.bat version

2. Verify output:
   - Should show version 2.0
   - Should show Banking Enterprise Edition
   - Should list features

3. Test compilation:
   ./run.sh compile banking_test_suite.cob -o banking.f90
   - Should compile successfully
   - Should create banking.f90

4. Test suite:
   ./run.sh test
   - Should run banking test suite
   - Should show success

TO TEST WEB DASHBOARD SCRIPTS:

1. Test web dashboard script:
   Linux/Mac: ./run-web.sh
   Windows:   run-web.bat

2. Verify output:
   - Should show startup banner

3. Test in browser:
   - Open http://127.0.0.1:8080
   - Should see professional dashboard
   - Should be able to click links

4. Stop server:
