                        BANKING CALCULATOR SPECIFICATIONS
================================================================================

INTEREST CALCULATOR:
  Formula: Interest = Principal × (Annual Rate / 100) × (Days / 365)

  Inputs:
    - Principal Amount (dollars)
    - Annual Rate (percentage)
    - Days (integer)

  Output:
    - Interest Earned (6 decimal precision)
    - New Balance

  Example:
    Principal: $10,000.00
    Rate: 1.5%
    Days: 30
    Result: $12.328767 interest

FEE CALCULATOR:
  Formula: Fee = Amount × (Fee Rate / 100)

  Inputs:
    - Amount (dollars)
    - Fee Rate (percentage)
    - Rounding Method (banker's, up, down)

  Output:
    - Fee Amount (2 decimal precision)
    - Final Amount

  Banker's Rounding:
    - 2.5 → 2 (nearest even)
    - 3.5 → 4 (nearest even)
    - Eliminates rounding bias

PRECISION ARITHMETIC TESTER:
  Operations: +, -, ×, ÷

  Inputs:
    - Value 1 (6 decimal places)
    - Operation
    - Value 2 (6 decimal places)

  Output:
    - Result (6 decimal precision)

  Validation:
    - Division by zero protection
    - Overflow detection
    - Precision maintained throughout

CTR THRESHOLD CHECKER:
  Threshold: $10,000.00 (Bank Secrecy Act)

  Input:
    - Transaction Amount

  Output:
    - CTR Required? (Yes/No)
    - Regulatory Guidance

  Compliance:
    - Automatic flagging
    - Regulatory citation
    - Clear visual indication

================================================================================
                        COMPLIANCE CHECKER SPECIFICATIONS
================================================================================

CHECK 1: PRECISION ARITHMETIC (SOX)
  Looks For: PIC clauses with V (decimal point)
  Example: PIC 9(10)V99
  Status: PASS if found, FAIL if missing
  Importance: Critical for Sarbanes-Oxley compliance

CHECK 2: LARGE TRANSACTION DETECTION ($10K+ CTR)
  Looks For: "10000", "CTR", "LARGE-TRANSACTION"
  Status: PASS if found, WARNING if missing
  Importance: Bank Secrecy Act requirement

CHECK 3: AUDIT TRAIL REQUIREMENTS
  Looks For: DISPLAY, WRITE, LOG statements
  Status: PASS if found, WARNING if missing
  Importance: SOX compliance, regulatory audit

CHECK 4: BANKER'S ROUNDING
  Looks For: ROUNDED clause
  Status: PASS if found, WARNING if missing
  Importance: Industry standard, eliminates bias

CHECK 5: ERROR HANDLING
  Looks For: ON SIZE ERROR, ON OVERFLOW
  Status: PASS if found, WARNING if missing
  Importance: Data integrity, prevent silent failures

CHECK 6: NAMING CONVENTIONS
  Looks For: Standard COBOL naming (A-Z, 0-9, -)
  Status: PASS if follows standard, WARNING otherwise
  Importance: Code maintainability, readability

RESULT CATEGORIES:
  - PASS: Meets requirement (green)
  - WARNING: Recommended but not critical (yellow)
  - FAIL: Does not meet requirement (red)

================================================================================
                        TROUBLESHOOTING
================================================================================

PROBLEM: Web dashboard won't load
SOLUTION:
  - Check web server is running (./run-web.sh or run-web.bat)
  - Verify port 8080 is not in use
  - Check firewall settings
  - Try http://localhost:8080 instead of 127.0.0.1

PROBLEM: API calls fail
SOLUTION:
  - Dashboard will fall back to demo mode
  - Check backend server is running
  - Verify API_BASE_URL in dashboard.js
  - Check browser console for errors

PROBLEM: Compilation doesn't work
SOLUTION:
  - In demo mode, mock Fortran is generated
  - For real compilation, ensure backend API is running
  - Check COBOL syntax is valid
  - Review browser console for errors

PROBLEM: Tests don't run
SOLUTION:
  - In demo mode, mock results are shown
  - For real tests, ensure backend API is running
  - Check test files exist in tests/ directory
  - Verify API endpoints are accessible

PROBLEM: Styling looks broken
SOLUTION:
  - Clear browser cache
  - Verify styles.css is loaded (check Network tab)
  - Check for console errors
  - Try different browser
